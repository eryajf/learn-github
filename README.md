&lt;h2&gt;本周话题：虚拟商品可以拉动 GDP&lt;/h2&gt;

&lt;p&gt;本周，脸书宣布&lt;a href=&quot;https://finance.sina.com.cn/roll/2022-06-22/doc-imizirau9930686.shtml&quot;&gt;推出&lt;/a&gt;线上的虚拟服装商店，里面有很多名牌服装（比如 Prada），用户买来在虚拟世界穿。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022062208.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;我看到这个新闻，不禁想到以前就思考过的一个观点：&lt;strong&gt;虚拟商品的交易，可以拉动 GDP 增长&lt;/strong&gt;，与实物商品的效果是一样的。&lt;/p&gt;

&lt;p&gt;虚拟服装、游戏道具、表情包......这些纯数字的虚拟商品，交易金额都是 GDP 的一部分，用户买得越多，GDP 就越高。不夸张地说，购买一个游戏道具，也是在为 GDP 做贡献。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022062113.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;有些朋友不了解经济学，我简单解释一下。GDP 有一种计算方法，就是统计一段时间内的消费总金额。消费金额越大，就表示经济总量越大，GDP 越高。至于消费的是什么，实体商品还是虚拟商品，并不区分。&lt;/p&gt;

&lt;p&gt;这里的关键是必须消费，才会计入 GDP。有一个经典例子，你付钱在餐馆吃饭，就会增加 GDP；你自己在家做饭，就不会，因为这样没有发生交易。&lt;/p&gt;

&lt;p&gt;这个原则也适用于虚拟物品。比如，微信打赏就不计入 GDP，因为它本质上是一种赠与，不是消费，也没有发生交易。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022062114.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;如果腾讯改变打赏规则，规定只能用 Q 币打赏，那就会计入 GDP。因为用户必须先购买 Q 币，这就会发生交易。&lt;/p&gt;

&lt;p&gt;同理，直播时给主播送礼物，是增加 GDP 的。以抖音直播为例，它使用&quot;抖币&quot;打赏，1元人民币可以购买10个&quot;抖币&quot;。用户将抖币换成礼物，送给主播。一束数字鲜花，需要10个抖币，抖音跟主播五五分成。抖音因此获得销售收入和利润，还必须为这些打赏交税。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022062115.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;总之，&lt;strong&gt;虚拟商品只要发生了交易，就会计入 GDP。&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;但是，虚拟商品与实物商品有一个重大区别：它不受资源约束，生产不消耗资源。举例来说，汽车的生产受到钢材、电力、土地、工人等许多资源的约束，不可能一下子增长太快；游戏道具就没有这个问题，想生产多少就有多少，根本就是零消耗。&lt;/p&gt;

&lt;p&gt;这意味着什么？&lt;/p&gt;

&lt;p&gt;&lt;strong&gt;虚拟商品可以在零消耗的情况下，增加 GDP。&lt;/strong&gt; 消费者购买的虚拟商品越多，GDP 就越高，而且不会增加对矿石、能源、粮食、交通运输等实物资源的消耗。&lt;/p&gt;

&lt;p&gt;这就为 GDP 增长提供了一种思路。当资源供应紧张或价格上涨时，实物商品的生产受到了约束；这时推动虚拟商品的交易，也能增加 GDP，促进国民经济的增长。&lt;/p&gt;

&lt;p&gt;最近，&lt;a href=&quot;http://www.xinhuanet.com/finance/2022-04/12/c_1128551710.htm&quot;&gt;游戏版号恢复发放&lt;/a&gt;，可能就有这方面的考虑。国家以后刺激经济的时候，也可以考虑刺激虚拟商品的消费，这完全不会增加实体经济的负担。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022062116.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;未来，人类社会进一步搬到互联网上，甚至出现元宇宙。那时，虚拟商品一定会越来越多，大部分 GDP 可能都来自虚拟商品的消费。&lt;/p&gt;

&lt;h2&gt;Go 语言学习资料包&lt;/h2&gt;

&lt;p&gt;前些天，一个朋友找我诉苦：&quot;Go 工程师太难找了。&quot;招聘广告挂了一个月，应聘者寥寥无几。&lt;/p&gt;

&lt;p&gt;他感到不理解，Go 的学习收益其实很高，为什么想招个合适的人却不容易呢。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202112/bg2021120101.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;要知道，Go 的势头这些年很好，国内应该已经是后端的第二大开发语言，仅次于 Java。&lt;/p&gt;

&lt;p&gt;越来越多的公司，整个后端都是用 Go 开发的，比如 B 站和七牛云。除了网络服务，Go 也是容器和微服务的首选语言，Docker 就是用 Go 写的。&lt;/p&gt;

&lt;p&gt;总之，Go 的招聘岗位一直在增加。虽然比不了 Java 和前端，但是关键一点是，后两者的学习者远多于 Go，就业竞争激烈，而 Go 工程师就没有竞争问题，更容易找到满意的工作。&lt;/p&gt;

&lt;p&gt;所以，学习 Go 的回报是很高的。何况 Go 语言入门其实不难，有 Java 和 C 语言基础的话，转 Go 是很容易的。&lt;/p&gt;

&lt;h2&gt;文摘&lt;/h2&gt;

&lt;p&gt;1、&lt;a href=&quot;https://zh.wikipedia.org/wiki/%E8%A5%BF%E7%A6%8F%E7%89%B9%E8%AE%A1%E5%88%92&quot;&gt;西福特计划&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;冷战时期，国际通信主要依靠海底电缆，以及大气的电离层反射无线电信号。&lt;/p&gt;

&lt;p&gt;美军非常担心，苏联会切断海底电缆，那就只有依靠不稳定的电离层，来联络海外部队了。美军就开始寻求，能稳定进行全球无线通信的方法。&lt;/p&gt;

&lt;p&gt;他们最终想出了一个异想天开的计划。马萨诸塞州西福特镇有一个卫星天线，美军决定在这个天线上方的电离层里面，散布4.3亿根铜针，每根长1.78厘米。所有铜针形成一个环绕地球的金属云圆环，以增强无线信号的发射，辅助远程通信。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202111/bg2021112113.jpg&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202111/bg2021112112.jpg&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;第一次散布铜针，发生在1961年10月21日，但是没有成功。很多铜针没有散布开，反而结成块，变成了太空垃圾。第二次散布发生在1963年5月9日，这次获得了成功，形成了一个由金属针组成的环状云轨道，并成功进行了通信。&lt;/p&gt;

&lt;p&gt;外界知道这件事情后，爆发了强烈的批评。另一个原因是通信卫星在这段时间发明了，所以铜针计划就没有再推进，最终被废弃。&lt;/p&gt;

&lt;p&gt;今天，它留下的铜针分布在高度为3500公里到3800公里之间的地球轨道上，偶然会有一些返回大气层。&lt;/p&gt;

&lt;h2&gt;本周话题：为什么软件变得复杂&lt;/h2&gt;

&lt;p&gt;我一直认为，软件开发的最大关注点，就是避免复杂性。软件设计越简单越好，太多的程序员以构建复杂的解决方案为荣。&lt;/p&gt;

&lt;p&gt;但是，本周有一篇文章让我反思，我的想法是不现实的：&lt;strong&gt;软件肯定会越变越复杂。&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;（一）&lt;/p&gt;

&lt;p&gt;这篇文章的作者是 Saleforce 公司的前端工程师诺拉·劳森（Nolan Lawson），题目就叫做&lt;a href=&quot;https://nolanlawson.com/2022/06/09/the-collapse-of-complex-software/&quot;&gt;《复杂软件的崩溃》&lt;/a&gt;（下图）。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022061407.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;大家可能知道，Salesforce 是一家世界级软件公司，专门开发企业软件，以产品复杂而闻名。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022061406.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;所以，这件事情很讽刺。一家出产复杂软件的公司，自家的著名程序员公开说，复杂软件会崩溃。&lt;/p&gt;

&lt;p&gt;当然，他在文章里面没提 Salesforce 的名字，但是怎么读都像在写亲身经历。下面摘录一段他的原文和配图，大家品味一下。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022061408.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;blockquote&gt;
  &lt;p&gt;&quot;任何在科技行业工作足够长的人，尤其是在大公司工作的人，都曾见过下面的场景。&lt;/p&gt;

&lt;p&gt;有一个遗留系统，很大很复杂，没有人完全理解它是如何工作的。&lt;/p&gt;

&lt;p&gt;架构师被要求&quot;修复&quot;系统，他们找来一块大的白板，把这个大系统分解成很多方框和箭头。对于遇到的问题，他们的解决方案就是......添加更多方框和箭头。没有任何一个可以从系统中消去，每个人都只是加上自己的那部分。&lt;/p&gt;
&lt;/blockquote&gt;

&lt;p&gt;这一段是不是写得很形象，就像是日常场景的描述。&lt;/p&gt;

&lt;p&gt;根据他的文章，我整理了一下，Saleforce 内部的情况大概是这样的。&lt;/p&gt;

&lt;blockquote&gt;
  &lt;ol start=&#39;1&#39;&gt;
&lt;li&gt;客户的需求非常复杂。为了满足这些需求，大型软件不可避免变得复杂。&lt;/li&gt;
&lt;li&gt;软件公司的管理者真正在意的不是系统的复杂性，而是利润。只要软件能赚钱，高层并不在意软件变得复杂。&lt;/li&gt;
&lt;li&gt;软件复杂性都落到少数架构师和高级程序员的头上。每个人加上自己的解决方案（方框和箭头），让软件越来越复杂，然后不可避免地，他们就会在一段时间后离开公司。&lt;/li&gt;
&lt;li&gt;复杂系统最终变得难以理解和维护，唯一的解决方法就是放弃旧系统，从头开始写一个新系统。&lt;/li&gt;
&lt;/ol&gt;
&lt;/blockquote&gt;

&lt;p&gt;所以，诺拉·劳森的结论很悲观：&lt;strong&gt;怎么解决软件的复杂性？解决不了。最后就是你走人，公司把软件推倒重来。&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;这就是一个大厂高级程序员的真实想法。国内的情况其实差不多，上面的描述完全适用于 BAT 内部的复杂系统。&lt;/p&gt;

&lt;p&gt;（二）&lt;/p&gt;

&lt;p&gt;诺拉·劳森还说了一个观点。大家通常认为，复杂系统往往会在经济繁荣的时候崩溃，因为业务太多，支撑不过来，但他认为不是这样的，&lt;strong&gt;系统崩溃往往发生在经济收缩期。&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;经济繁荣时期，软件公司会大量雇佣新员工，投入更多的财力和人力，支撑复杂系统。等到经济收缩期，公司开始减少投入、冻结招聘或裁员，复杂系统可能就会在这个时候出问题，变得难以维护。&lt;/p&gt;

&lt;p&gt;现在就是经济收缩期，那么接下来，会不会就是软件故障的高发期，我们将看到很多复杂系统的崩溃？&lt;/p&gt;

&lt;h2&gt;前端高频面试题（2022版）&lt;/h2&gt;

&lt;p&gt;这两年，客观地说，前端开发的热度有所下降。主要原因是前端技术逐步稳定，以及智能手机的普及度见顶了。&lt;/p&gt;

&lt;p&gt;但是，前端依然是 IT 行业中最活跃的分支。一年一度的 &lt;a href=&quot;https://octoverse.github.com/&quot;&gt;GitHub 调查&lt;/a&gt;中，JavaScript 多年来一直稳居第一，TypeScript 更是快速上升到今年的第四位，仅次于 Python 和 Java。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022061616.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;技术稳定带来的一个后果，就是新框架、新工具少了，大家更关注已有框架/工具的改进和功能增加。&lt;/p&gt;

&lt;p&gt;大厂的前端团队也转向了精细化探索，注重如何做得更细、更好，垂直化的技术领域（比如可视化、工程化等）得到了更多的关注。企业的用人要求也越来越高，从能够上手工作就可以要人，变成了要求深入某个领域。&lt;/p&gt;

&lt;p&gt;总的来说，&lt;ins&gt;现在的前端开发处在下图右侧的那个椭圆，比前一个阶段的难度上升了&lt;/ins&gt;。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022061617.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;1、&lt;a href=&quot;https://gizmodo.com/vr-researches-simulate-kisses-with-ultrasonic-transduce-1848849489&quot;&gt;虚拟接吻&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/220204/bg2022043003.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;卡内基梅隆大学开发了一种新技术，让 VR 头盔的佩戴者可以感受到虚拟接吻。具体做法是，头盔向嘴唇发射超声波，里面包含了一些微小颗粒，让嘴唇、牙齿甚至舌头产生触感。&lt;/p&gt;

&lt;p&gt;上图可以看到，VR 头盔的下方安装了一排超声波发生器，对准红点的位置发射。科学家开玩笑，可以用它开发&quot;接吻机&quot;。&lt;/p&gt;

&lt;p&gt;除了产生嘴唇的触感，这种技术还可以在游戏中，让用户感受到风拂过脸部、蜘蛛网碰到脸，甚至食物和饮料进入嘴里的虚拟感受。&lt;/p&gt;

&lt;p&gt;上图是一只巨大的虚拟蜘蛛将大量毒药倾泻到用户身上，用户可以感觉到毒药溅到嘴唇上。&lt;/p&gt;

&lt;p&gt;2、&lt;a href=&quot;http://www.lanxiongsports.com/posts/view/id/22687.html&quot;&gt;LED 篮球场&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;传统的篮球场都是木地板，木头具有弹性，可以吸收震动，保护球员。现在，最新的 LED 玻璃也已经能够提供足够的弹性。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060808.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;并且，它自身能发光，可以作为显示屏，实时显示比赛数据，并且配合球场活动，营造气氛。在黑暗中，界线依然保持醒目。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060809.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;下面是字母 a 和 o 在五年中的变化。&lt;/p&gt;

&lt;p&gt;这些字体应用到印刷品，就是下面的样子。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022050305.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;h2&gt;文摘&lt;/h2&gt;

&lt;p&gt;1、&lt;a href=&quot;https://www.ifanr.com/app/1448161&quot;&gt;如何增加牙膏的销量&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;1950年代，一家国外的牙膏公司，向公众征求能够大幅提高销售额的点子。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202202/bg2022020805.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;几天之后，有一个人来应征，声称他有办法让销售额快速增长40%，而且实施起来不需要很大成本。他要价10万美元。&lt;/p&gt;

&lt;p&gt;公司管理层犹豫了数周，最后还是同意了给钱。&lt;/p&gt;

&lt;p&gt;等到法律手续和付款流程完成后，这个人给出装有一张小纸条的信封，小纸条上只有四个英文单词：&lt;/p&gt;

&lt;blockquote&gt;
  &lt;p&gt;&quot;Make The Hole Bigger&quot;&lt;/p&gt;
&lt;/blockquote&gt;

&lt;p&gt;这句话翻译成中文，就是&quot;让牙膏开口更大一点&quot;。&lt;/p&gt;

&lt;p&gt;此前，管状牙膏的开口一般是5毫米直径。稍加计算就可以知道，当直径从5毫米增加到6毫米时，假设挤出的牙膏长度不变，挤出量会增加44%。&lt;/p&gt;

&lt;p&gt;原先顾客用一管牙膏的时间，现在要1.4管牙膏才能满足需求。看似很小的改变，却刷新了这家公司的销售记录，创造了历史。&lt;/p&gt;

&lt;h2&gt;本周话题：程序员是怎样的人&lt;/h2&gt;

&lt;p&gt;我最近看到一个令人深思的心理学实验，叫做&quot;&lt;a href=&quot;https://moxie.org/2013/01/07/career-advice.html&quot;&gt;斯坦福监狱实验&lt;/a&gt;&quot;（Stanford Prison Experiment），跟大家分享一下。&lt;/p&gt;

&lt;p&gt;1971年，一个美国的心理学家在斯坦福大学心理学系的地下室，搭建了一个模拟监狱。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060818.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;他找来一批志愿者，让他们掷硬币，一半人扮演囚犯，另一半人扮演狱警。这些志愿者都是正常人，从来没有接触过监狱，心理学家也不告诉他们，该怎么扮演囚犯和狱警。&lt;/p&gt;

&lt;p&gt;每个人换上角色服装，进入逼真的环境。一开始，大家都举止笨拙，不知道做什么，还觉得这件事有点滑稽。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060821.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;但是过了一会，事情就开始发生变化。囚犯们慢慢摆脱了从前的身份，开始接受现在的角色，站在囚犯的立场思考。他们感受到心理压力，没心思吃东西，向&quot;狱警&quot;提出抗议，彼此互相指责。&lt;/p&gt;

&lt;p&gt;狱警们也开始像真正的狱警，管起了囚犯，划定单人牢房，让囚犯呆在里面，接受心理惩罚。同时，对囚犯发表长篇大论的说教，听任囚犯互相攻击，还会缓慢地在走​​廊上踱步，用木棍敲打牢房门的栅栏。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060820.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;最惊人的是，过了两天，心理学家让所有人调换角色，囚犯扮演狱警，狱警扮演囚犯。结果，双方很快就适应了新角色，行为变得与新角色一致。&lt;/p&gt;

&lt;p&gt;由于所有人太入戏，心理学家担心发生难以控制的局面，原定两周的实验不得不在第六天就宣布结束。&lt;/p&gt;

&lt;p&gt;心理学家从这个实验，得到一个结论：&lt;strong&gt;人类行为受到工作环境的强烈影响，你会变成环境要求你成为的样子。&lt;/strong&gt; 正常人关进监狱，就会变得囚犯般俯首听命，暗中算计；要是当上狱警，就会不由自主地居高临下、残忍霸道。&lt;/p&gt;

&lt;p&gt;这个实验带给我们的启示就是，&lt;strong&gt;要小心你的工作，它会改变你，让你成为另一个人。&lt;/strong&gt; 很多人都有一种印象，会计师的性格是斤斤计较、对数字格外敏感，律师的性格是重视利益最大化，海关执法人员的性格则是充满怀疑、不信任他人，这就是职业对性格的改变。&lt;/p&gt;

&lt;p&gt;这不禁让我思考一个问题：程序员这个职业，会如何改变一个人的性格？&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060822.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;我能想到的有下面几点。&lt;/p&gt;

&lt;p&gt;&lt;ins&gt;（1）你会变得习惯孤军奋斗，靠自己解决问题。&lt;/ins&gt;编程本质上属于个人劳动，不是团队劳动。一段代码终究需要你一个人写出来，不可能两个人合写一个函数，你写上半段，我写下半段。现代软件工程是基于分工的协作，每个人完成自己那份工作，然后才能互相协作。&lt;/p&gt;

&lt;p&gt;&lt;ins&gt;（2）你会习惯孤独。&lt;/ins&gt;编程非常孤独，日复一日、长时间地与机器打交道，很少与他人面对面交流。如果遇到赶工期，一连好几天，都说不了几句话。我能想到的类似工作，大概只有作家。&lt;/p&gt;

&lt;p&gt;&lt;ins&gt;（3）你必须非常细致，做到谨小慎微。&lt;/ins&gt;每个细节都要考虑到，任何一个字母、一个小数点都不能写错，每个逻辑条件都要理清。只要有一个疏忽，代码就会无法运行，或者留下安全隐患。&lt;/p&gt;

&lt;p&gt;&lt;ins&gt;（4）强大的心理承受能力。&lt;/ins&gt;程序员是压力很大的工作，公司越大，压力越大，代码写得慢、写错了、或者写不出来，你都要承担责任。有时候出现重大线上 Bug，别人在屁股后面催着你，你却迟迟想不出问题出在哪里，又没有办法测试，那种心理压力真会让人崩溃。&lt;/p&gt;

&lt;p&gt;总之，一个人如果没有形成稳定的个性之前，就去当职业程序员，可能就会变成上面这样：不习惯与他人紧密协同，喜欢单干，长时间的孤独劳动，非常重视细节，任何一处都不能偷懒或写错，心理压力巨大。&lt;/p&gt;

&lt;p&gt;如果你想好了，能够接受变成这样的人，可以来当程序员。&lt;/p&gt;

&lt;h2&gt;个人如何提高抗风险能力&lt;/h2&gt;

&lt;p&gt;端午节前，孙明展老师联系我，他们谱蓝公司要搞下半年的活动了，希望周刊发一个消息。&lt;/p&gt;

&lt;p&gt;老读者可能知道，孙老师是金融业人士，从业30多年。他从这个周刊创刊开始，就一直很支持我，所以虽然这里以计算机内容为主，我还是愿意向大家介绍他的公司。&lt;/p&gt;

&lt;p&gt;他是广州中山大学统计系硕士毕业，进入世界500强保险公司，后来辞职创业，创办了财务咨询品牌&quot;谱蓝&quot;。公司很成功，已经创办超过10年，前不久完成了 B 轮融资。他们的主要业务是财务咨询和财务服务，大家可以访问&lt;a href=&quot;http://www.trussan.com/about#company_detail&quot;&gt;官网&lt;/a&gt;，或者网上搜索&quot;谱蓝&quot;。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060401.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;他在业余时间还担任中山大学金融系、统计系专业（兼职）硕士导师，并且坚持在&lt;ins&gt;微信公号《孙明展》&lt;/ins&gt;做财务知识科普。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060403.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;过去一年，宏观经济发生剧烈变化，很多行业受到严重影响，比如零售、教培、互联网等等。所以，谱蓝公司这次的活动主题就是 &lt;strong&gt;&quot;个人如何提高抗风险能力&quot;&lt;/strong&gt;，为大家在财务上出谋划策，当然也是为了推广企业品牌。&lt;/p&gt;

&lt;p&gt;提高抗风险能力，说来很简单，只需要做到三件事。&lt;/p&gt;

&lt;blockquote&gt;
  &lt;ol start=&#39;1&#39;&gt;
&lt;li&gt;&lt;ins&gt;有稳定收入&lt;/ins&gt;，通常来自工资或资产。&lt;/li&gt;
&lt;li&gt;&lt;ins&gt;防范风险&lt;/ins&gt;，预留好合理现金流。&lt;/li&gt;
&lt;li&gt;&lt;ins&gt;配置资产&lt;/ins&gt;，让现有财产保值增值。&lt;/li&gt;
&lt;/ol&gt;
&lt;/blockquote&gt;

&lt;p&gt;但是，上面每一点做起来都不容易。现在的金融产品都设计得非常复杂，不管是基金还是保险，合同里面都有长长的免责条款，普通人怎么知道应该选择哪一种。&lt;/p&gt;

&lt;p&gt;这次的咨询活动，就是由谱蓝的理财师为大家一对一服务，你可以得到专业人士的指导，&lt;ins&gt;如何防范风险和资产配置，提高个人财务状况的安全性&lt;/ins&gt;。&lt;/p&gt;

&lt;p&gt;首先，他们会根据你们家庭的财务状况，构建家庭资产负债表，进行财务分析，指出风险漏洞。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060406.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;然后，他们会为你给出一整套家庭资产配置方案，包括存款、投资、保险等金融产品的百分比，兼顾抗风险和养老的需求，尽量做到保值增值。如果是已婚有孩子，还会考虑为孩子做教育金储蓄规划。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060405.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;你拿到的资产配置方案，会包含多个部分（家庭保障规划、教育金规划、养老金规划......）。理财师保证专业和中立，你提任何问题，都会得到详细讲解。&lt;/p&gt;

&lt;p&gt;周刊读者可以免费参加这一次谱蓝的资产配置规划服务，前100名读者还可以获赠一份《家庭财务宝典》。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060407.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;微信扫描上方二维码，就可以了解活动详情，并报名参加。限量200人，领完即止。&lt;/p&gt;

&lt;h2&gt;科技动态&lt;/h2&gt;

&lt;p&gt;1、&lt;a href=&quot;https://www.businessinsider.com/sony-patents-new-system-converts-bananas-into-game-playstation-controllers-2021-3&quot;&gt;香蕉遥控器&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202203/bg2022032014.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;专利文件使用香蕉举例，用户可以拿着一只香蕉，在上面按下虚拟按键，操纵游戏。文件说&quot;改用橙子、咖啡杯和钢笔都可以&quot;。&lt;/p&gt;

&lt;p&gt;2、&lt;a href=&quot;http://www.stdaily.com/index/kejixinwen/202205/43bff4a13a274dc6a6f31ff9abd0baf7.shtml&quot;&gt;远海养殖船&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022053101.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022053102.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;有些名贵的食用鱼（大黄鱼、石斑鱼、鲑鱼等）不适合近海养殖，一家青岛公司就制造了全球最大的10万吨养殖船&quot;国信1号&quot;，开到远海，在船里养鱼。船的长度为250米，鱼的年产量预计为3700吨。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022053103.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022053104.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;这艘船可以一刻不停地用海水替换养殖用水，保证了水质，也就保证了鱼的品质，还能避开台风、赤潮等自然灾害。养殖舱内的水、氧、光、饲都是集中控制。该公司透露，国信2号和国信3号将在两年内交付使用。&lt;/p&gt;

&lt;p&gt;3、&lt;a href=&quot;https://www.tsinghua.edu.cn/info/1175/95080.htm&quot;&gt;管道机器人&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060305.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;清华大学机械系发明了一种微型管道机器人，直径小于1厘米，长度4.7厘米，重量2.2克。这么小的机器人，内部集成了运动机械、电池、摄像头和通信模块。&lt;/p&gt;

&lt;p&gt;它可以接收外部指令，前进和转弯，速度大约每秒5厘米，还可以弯曲成各种形状。它的主要用途是航空发动机的管道检修。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060304.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;4、&lt;a href=&quot;https://www.europarl.europa.eu/news/en/press-room/20220603IPR32196/deal-on-common-charger-reducing-hassle-for-consumers-and-curbing-e-waste&quot;&gt;欧盟统一充电器&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060805.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;本周，欧盟通过了一个法律草案，规定到2024年秋天，欧盟销售的所有中小型便携式电子设备都必须使用 USB Type-C 充电口，包括手机、平板、阅读器、耳塞、数码相机、耳机、手持游戏机和便携式扬声器等。笔记本电脑有40个月的缓冲期，最终也要改成 Type-C 充电。&lt;/p&gt;

&lt;p&gt;这是为了减少电子垃圾，减轻消费者负担。据估计，欧盟每年废弃的充电器约11,000吨，一旦统一充电器，消费者可以每年节省高达2.5亿欧元。&lt;/p&gt;

&lt;p&gt;受影响最大的将是苹果手机，不过该法律不涉及无线充电，苹果公司可以选择彻底取消 iPhone 的有线充电。&lt;/p&gt;

&lt;h2&gt;文摘&lt;/h2&gt;

&lt;p&gt;1、&lt;a href=&quot;https://nodum.org/10-reasons-for-electric-tanks/&quot;&gt;坦克可以电动吗？&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;坦克是战场的庞然大物，要求坚实耐用、能在艰苦条件下作战。这使得大多数人相信，坦克不适合电动化。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202202/bg2022020613.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;这种想法是不对的，电力其实很适合作为坦克的能源。&lt;/p&gt;

&lt;p&gt;（1）军营里面不可能制造燃料，但可以发电。未来便携式小型核反应堆一旦技术成熟，部队就可以把核反应堆带到任何地点，利用核电每晚为电池充电。&lt;/p&gt;

&lt;p&gt;（2）一个陆军装甲师前进时，每天需要多达200万升燃料。运送这样大量的燃料，是一个非常巨大的后勤挑战。特别是在战时，敌人会重点袭击补给车队。&lt;/p&gt;

&lt;p&gt;（3）电动车的零件较少，维护就比较简单：无需换油，更少的发动机零件，更少的部件磨损等等。这也简化了战场的物流。&lt;/p&gt;

&lt;p&gt;（4）电气系统是模块化的，而且往往是通用的。这意味着，可以快速更换或更新模块。另外，一辆车可以有几个电动机和几个单独的电池仓，坏了一个不会影响整车的工作，汽油车就做不到。&lt;/p&gt;

&lt;p&gt;（5）电动车的加速比内燃机快得多，这在战斗时肯定会有所帮助。&lt;/p&gt;

&lt;p&gt;（6）电机可以用作移动电源，为其他工具甚至武器提供能量。&lt;/p&gt;

&lt;p&gt;（7）电动坦克由计算机控制，能根据不同情况，采用不同的模式，比如公路模式、越野模式、节能模式等等。&lt;/p&gt;

&lt;p&gt;（8）与内燃机不同，电动机不需要消耗空气，这可以让电动坦克在水中完全淹没时继续前进。&lt;/p&gt;

&lt;p&gt;（9）电动车易于隐蔽，不易被发现。它们不会产生那么多热量，也没有排气系统，这使得热像仪难以注意到它们。它们行动起来也更安静，没有噪音。&lt;/p&gt;

&lt;p&gt;上面是电动坦克的优点，但还有一些技术难题等待克服。除了如何在战场上充电这个问题，主要是下面两个问题：首先，需要安全性更高的固态电池，这种电池在枪击或爆炸时，都不会起火；其次，需要降低成本，电动坦克的成本还是太高，远远高于柴油坦克。&lt;/p&gt;

&lt;p&gt;尽管如此，可以肯定的是，军用的电动机器在不久的将来肯定会越来越多。现在已经出现了电动的战斗机器人、电动卡车、电动无人机，那么更易于维护、更快、更隐蔽的电动坦克迟早也将出现。&lt;/p&gt;

- 💫 [字节序探析：大端与小端的比较](http://www.ruanyifeng.com/blog/2022/06/endianness-analysis.html) 
  > &lt;p&gt;今天谈谈一个重要的计算机概念，大家可能都听说过它，但是很少深究，那就是字节序（Endianness）。&lt;/p&gt;
&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060101.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;h2&gt;一、概念&lt;/h2&gt;

&lt;p&gt;字节序指的是，&lt;strong&gt;多字节数据的内存排列顺序&lt;/strong&gt;。这样说比较抽象，使用图形解释就很好懂。&lt;/p&gt;

&lt;p&gt;内存好比一排房间，每个字节是一间房。每间房都有门牌号（内存地址），从0号开始，然后是1号、2号......&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060102.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;0号字节的地址小，称为低位内存；3号字节的地址大，称为高位内存。&lt;/p&gt;

&lt;p&gt;现在有一个数值&lt;code&gt;abcd&lt;/code&gt;要放进这些房间，每个房间放一个数字，那么有两种放法。&lt;/p&gt;

&lt;p&gt;第一种放法是，第一位&lt;code&gt;a&lt;/code&gt;放在低位地址（0号），最后一位&lt;code&gt;d&lt;/code&gt;放在高位地址（3号）。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060103.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;这种排列称为&quot;&lt;strong&gt;大端序&lt;/strong&gt;&quot;（big-endian，简称 BE），即大头在前，因为&lt;code&gt;a&lt;/code&gt;是&lt;code&gt;abcd&lt;/code&gt;的大头（最重要的数字）。&lt;/p&gt;

&lt;p&gt;第二种放法是，第一位&lt;code&gt;a&lt;/code&gt;放在高位地址（3号地址），最后一位&lt;code&gt;d&lt;/code&gt;放在低位地址（0号地址）。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060104.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;这种排列称为&quot;&lt;strong&gt;小端序&lt;/strong&gt;&quot;（little-endian，简称 LE），即小头&lt;code&gt;d&lt;/code&gt;在前。&lt;/p&gt;

&lt;p&gt;大端序和小端序合称字节序，这两个名字来自18世纪的英国小说《格列佛游记》。某国分成两派，一派认为鸡蛋应该从大头吃起，称为&quot;大端派&quot;；另一派认为，鸡蛋应该从小头吃起，称为&quot;小端派&quot;。两派相执不下，谁也无法说服谁，最后甚至为此交战。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060105.jpg&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;h2&gt;二、可读性&lt;/h2&gt;

&lt;p&gt;对于人类来说，不同字节序的可读性是不一样的。大部分国家的阅读习惯是从左到右阅读。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060106.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;大端序的最高位在左边，最低位在右边，符合阅读习惯。所以，对于这些国家的人来说，&lt;strong&gt;从左到右的大端序的可读性更好。&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;但是现实中，从右到左的小端序虽然可读性差，但应用更广泛，x86 和 ARM 这两种 CPU 架构都采用小端序，这是为什么？&lt;/p&gt;

&lt;p&gt;或者换一种问法，两种不同的字节序为什么会并存，统一规定只使用一种，难道不是更方便吗？&lt;/p&gt;

&lt;p&gt;原因是它们有各自的适用场景，某些场景大端序有优势，另一些场景小端序有优势，下面就逐一分析。&lt;/p&gt;

&lt;h2&gt;三、检查奇偶性&lt;/h2&gt;

&lt;p&gt;小端序优势最明显的，大概就是检查奇偶性，即通过查看个位数，确定某个数字是奇数还是偶数。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060112.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;以&lt;code&gt;123456&lt;/code&gt;为例，大端序从左到右排列，计算机必须一直读到最后一位的个位数&lt;code&gt;6&lt;/code&gt;，才能确定这是偶数。&lt;/p&gt;

&lt;p&gt;小端序是从右到左排列，个位数在第一位。所以，只要读取第一位，就能确定它是偶数。&lt;/p&gt;

&lt;h2&gt;四、检查正负号&lt;/h2&gt;

&lt;p&gt;一个类似的场景是检查正负号，确定一个数是正数还是负数。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060113.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;大端序的符号位在左边第一位，小端序的符号位在右边最后一位。所以，大端序有优势，只看第一位就能知道是不是负数。&lt;/p&gt;

&lt;h2&gt;五、比较大小&lt;/h2&gt;

&lt;p&gt;下一个操作是比较大小。现在有三个数字，需要比较大小：43662576，594，2。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060108.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;上图是大端序排列，因为是从左到右排列，所以三个数字在右边个位数对齐。比较大小时，计算机就不得不读取每一个数的所有位，直到个位数，再进行比较。&lt;/p&gt;

&lt;p&gt;如果改成小端序，就是下面的排列方式。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060109.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;小端序是从右到左，所以三个数字在第一位对齐。计算机就不需要读取所有位，哪个数字先读不到下一位，就是最小的。比如，&lt;code&gt;2&lt;/code&gt;这个数字就没有第二位，所以读到第二位时，就知道它是最小的。&lt;/p&gt;

&lt;p&gt;所以，比较大小时，小端序有优势。&lt;/p&gt;

&lt;h2&gt;六、乘法&lt;/h2&gt;

&lt;p&gt;接下来，再看乘法操作。&lt;/p&gt;

&lt;p&gt;乘法是逐位相乘，每一轮乘法都要向前进位。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060110.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;上图是大端序的24165乘以3841。大端序的乘法是向左进位，也就是向左边扩展，必须等到每一轮的结果都出来（上例是四轮），再相加统一写入内存。&lt;/p&gt;

&lt;p&gt;如果改成小端序的乘法，就不需要等待下一轮的结果，每一轮都可以直接写入内存。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060111.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;上图是小端序的24165乘以3841。小端序的乘法是向右进位，也就是向右边扩展，左边的边界不变。每一轮结果写入内存后，就不需要移动，后面有变化只需要改动对应的位就行了。&lt;/p&gt;

&lt;p&gt;因此，小端序的乘法有明显优势。&lt;/p&gt;

&lt;h2&gt;七、任意精度整数&lt;/h2&gt;

&lt;p&gt;上一个例子的从低位开始计算的特性，对于任意精度整数特别有用。任意精度整数又称大整数，可以存放任意大小的整数。&lt;/p&gt;

&lt;p&gt;它的内部实现是把整数分成一个个较小的单位，通常是 uint32（无符号32位整数）或 uint64（无符号64位整数），按顺序组合在一起。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060115.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;如果是大端序，第一个 u64 就是这个整数最大的部分。运算时，一旦这个数发生变化，需要进位，后面的所有位都必须移动和改写。小端序发生进位时，往往就不需要所有位移动。&lt;/p&gt;

&lt;p&gt;小端序的另一个好处是，如果逐字节的运算从个位数开始（比如乘法和加法），可以从左到右依次运算一个个 u64，算完上一个再读取下一个。大端序就不行，必须读取整个数以后再进行运算。&lt;/p&gt;

&lt;h2&gt;八、更改类型&lt;/h2&gt;

&lt;p&gt;最后一个例子是，C 语言有一种 cast 操作，可以强制改变变量的数据类型，比如把32位整数强行改变为16位整数。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202206/bg2022060114.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;上图中，32位整数&lt;code&gt;0x00000001&lt;/code&gt;更改为16位整数&lt;code&gt;0x0001&lt;/code&gt;，大端序是截去前面两个字节，这时指向这个地址的指针必须向后移动两个字节。&lt;/p&gt;

&lt;p&gt;小端序就没有这个问题，截去的是后面两个字节，第一位的地址是不变的，所以指针不需要移动。&lt;/p&gt;

&lt;h2&gt;九、总结&lt;/h2&gt;

&lt;p&gt;综上所述，大端序和小端序各自的优势如下。&lt;/p&gt;

&lt;p&gt;如果需要逐位运算，或者需要到从个位数开始运算，都是小端序占优势。反之，如果运算只涉及到高位，或者数据的可读性比较重要，则是大端序占优势。&lt;/p&gt;

&lt;h2&gt;本周话题：晋升制度的问题&lt;/h2&gt;

&lt;p&gt;我最近读到&lt;a href=&quot;https://www.warp.dev/blog/problems-with-promotion-oriented-cultures&quot;&gt;一篇文章&lt;/a&gt;，作者是谷歌表格（Google Sheets）的前主管 Zach Lloyd，现在已经离职创业了。他说自己烦透了谷歌的晋升制度。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022052405.jpg&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;我读完发现，谷歌的晋升制度跟国内大厂一模一样，员工行为也是晋升导向，产生的问题也是相似的，所以这篇文章读起来根本不像在说国外的事情。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022052210.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;现在正好是晋升季，很多人忙了一年，等的就是这一天。互联网大厂的财年三月底结束，然后主管提名晋升人选，四五月进行晋升答辩。只要通过了，你的职级就会上调，工资、期权、职务、地位都会随之增加。&lt;ins&gt;如果你连续好几年没有获得晋升，就是一个危险的信号&lt;/ins&gt;，表明公司并不肯定你这几年的工作。&lt;/p&gt;

&lt;p&gt;所以毫不奇怪，&lt;strong&gt;大厂员工的行为都是晋升导向&lt;/strong&gt;。做任何事情之前，都会首先考虑是否有利于晋升。即使你真的非常关心其他事情，也不得不权衡它们对晋升的影响。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022052211.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;Zach Lloyd 说，这样的制度设计会导致4个严重的问题。我觉得分析得非常好，下面跟大家分享。&lt;/p&gt;

&lt;p&gt;（1）晋升导向的最大问题是，&lt;strong&gt;晋升标准与公司利益并不完全一致&lt;/strong&gt;，员工追求晋升和短期绩效，可能不会导致公司利益最大化，甚至会伤害公司的长期利益。&lt;/p&gt;

&lt;p&gt;让我们看一个例子。假定有两个程序员，甲专注开发新产品新功能，乙专注维护老产品（修补 bug、提高性能、优化算法、升级版本、撰写文档、充当客服等等），请问谁会得到晋升？&lt;/p&gt;

&lt;p&gt;通常情况是，甲会得到晋升，乙很难晋升。因为维护老产品一般不产生绩效，有巨大的晋升劣势。这有两个原因，一是老产品对业务的促进作用，在前期上线时已经被考虑在内了，后续的维护只是兑现当初的预期，维护得再努力也很难超出预期；二是难以计量维护者的贡献，老产品取得成功，到底是因为设计和开发得好，还是因为维护得好？&lt;/p&gt;

&lt;p&gt;因此，&lt;ins&gt;晋升导向鼓励程序员重视新产品新功能，轻视老产品的维护。&lt;/ins&gt;这在现实中屡见不鲜，大厂的软件喜欢追求大而全，喜欢造轮子，创造大同小异的竞品，而老产品缺乏维护，原因就在这里。没有什么比自己的代码在关键业务落地，更重要了，这样几乎肯定会得到晋升，其他都是次要考虑。&lt;/p&gt;

&lt;p&gt;（2）&lt;strong&gt;晋升导向强化了等级制&lt;/strong&gt;，在公司中引入过多的层级。晋升文化越强的公司，必然伴随复杂的内部组织架构，各团队之间的协调也会越困难。&lt;/p&gt;

&lt;p&gt;（3）&lt;strong&gt;晋升导向人为制造了一个气氛很可怕的&quot;晋升季&quot;。&lt;/strong&gt; 为了得到晋升提名，容易产生内部矛盾，甚至勾心斗角。获得提名的人要浪费大量时间，制作 PPT，准备晋升答辩，这会降低生产力，增加公司成本。&lt;/p&gt;

&lt;p&gt;（4）&lt;strong&gt;晋升导向造成一种不良的公司文化。&lt;/strong&gt; 晋升至上使得那些真正关心产品、关心代码健康的工程师士气低落，有时不得不在&quot;对用户有利&quot; vs &quot;对晋升有利&quot;之间做出痛苦的选择。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022052406.jpg&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;上面就是晋升导向的四个缺点，那么这是否意味着，它就是一个很糟糕的制度呢？&lt;/p&gt;

&lt;p&gt;Zach Lloyd 认为，也不能这么说。晋升导向的优点是，对员工会产生巨大的考核和激励，其他制度的效果恐怕都不如它。&lt;/p&gt;

&lt;p&gt;因此他建议，最好采取一些措施，让员工的个人利益与公司利益尽量一致，比如强调用户优先，再比如聘请不是一味追求晋升的工程师。&lt;/p&gt;

&lt;p&gt;我认为，这些措施都不现实，唯一比较可行的是，当公司大到一定程度以后，就把一些部门和团队拆分出去，成为独立的创业公司，而且最好这些创业公司之间有竞争关系。对于小公司来说，员工都明白，只有公司取得成功，个人才能取得成功，公司利益和个人利益是一致的。他们就会更关心公司会不会倒闭，而不是自己的晋升。&lt;/p&gt;

&lt;p&gt;1、&lt;a href=&quot;https://jamessevedge.com/articles/red-light-green-light/&quot;&gt;红绿灯的故事&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022052610.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;有一次，我开车载着三岁的儿子，他叫本。&lt;/p&gt;

&lt;p&gt;我们在十字路口的红绿灯处停下，我说：&quot;嘿，本，交通灯是什么颜色？&quot; &lt;/p&gt;

&lt;p&gt;他说，&quot;它是绿色的。&quot; &lt;/p&gt;

&lt;p&gt;我纠正他：&quot;本，交通灯现在是停车标记，这是什么颜色？好好看看。&quot; 他说：&quot;绿色的。&quot;&lt;/p&gt;

&lt;p&gt;过了一会，交通灯变了。他说：&quot;灯变红了，我们可以走了。&quot;&lt;/p&gt;

&lt;p&gt;我第一个想法是完蛋了，这孩子是色盲。但是转念一想，不对啊，色盲也不会把绿灯看成红灯，红灯看成绿灯啊。&lt;/p&gt;

&lt;p&gt;回家后，我拿出图片，让他分辨红色和绿色，他每一次都做对了。但是后来又经过十字路口，他还是认错红绿灯，我非常困惑。&lt;/p&gt;

&lt;p&gt;过了三周，我岳母来看我们，我们一起坐车出去。我坐在后排，就在本的儿童座椅旁边。我们在一个交通灯前停下，我突然明白是怎么回事了。本坐在后排的儿童座椅上，前排座位挡住了他的视线，他看不见正前方的交通灯，他看到的是十字路口侧边的交通灯，它的颜色正好跟正前方的相反。&lt;/p&gt;

&lt;p&gt;这件事让我明白了一个事实，你的位置决定了你所看到的。当你坐在前排，实际做事时，你看到的东西、你的视野是后排的人所没有的。如果后排的人说一些没有意义的话，你就忽略它，并继续前进。&lt;/p&gt;

&lt;h2&gt;本周话题：汽车行业的顶峰可能过去了&lt;/h2&gt;

&lt;p&gt;4月份的时候，小鹏汽车的董事长&lt;a href=&quot;https://finance.sina.cn/2022-04-14/detail-imcwipii4318205.d.html&quot;&gt;何小鹏&lt;/a&gt;说，如果上海和周边的供应链企业一直无法复工，五月份可能中国所有的整车厂都要停工停产了。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022051613.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;现在已经快要5月底了，上海还没有恢复常态，中国汽车业今年大概很困难了。我看到的&lt;a href=&quot;https://mp.weixin.qq.com/s/MWe8HHmjaD3jEPAB7SOIuA&quot;&gt;最新统计&lt;/a&gt;是，四月份的中国汽车总产量和总销量，比去年同月接近腰斩。&lt;/p&gt;

&lt;p&gt;虽然这是疫情造成的，但我觉得，就算疫情过去了，汽车行业恐怕也不会走向繁荣，&lt;strong&gt;行业顶峰可能已经过去了。&lt;/strong&gt;&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022051614.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;大家猜猜看，中国汽车销售最多的是哪一年？&lt;/p&gt;

&lt;p&gt;答案是五年前的2017年，当年总共销售了近2900万辆，以后再也没有超过这个数字。2020年下降到2531万辆，去年2021年有所反弹，达到2627万辆。&lt;/p&gt;

&lt;p&gt;中国经济一直在增长，为什么&lt;a href=&quot;https://m.21jingji.com/article/20220112/herald/1accb64f68d2d8a44bfe7008ab41707b.html&quot;&gt;汽车销量&lt;/a&gt;不增长了，2900万辆可能就到顶了？&lt;/p&gt;

&lt;p&gt;参考西方国家的情况，汽车销量都是到了一定阶段后，就无法再增长了。&lt;a href=&quot;https://www.statista.com/statistics/199983/us-vehicle-sales-since-1951/&quot;&gt;美国&lt;/a&gt;在2000年销售了1700万辆汽车，以后就停滞不前了，去年的销量不到1500万辆。&lt;/p&gt;

&lt;p&gt;而且，所有西方发达国家（美国、澳大利亚、比利时、德国、意大利、日本、新西兰等等）有&lt;a href=&quot;https://bigthink.com/the-present/peak-car/&quot;&gt;两个重要指标&lt;/a&gt;----每辆车每年的行驶里程数，以及人均每年开车出行距离----从2004年后都在不断下降。&lt;/p&gt;

&lt;p&gt;这说明，&lt;strong&gt;汽车拥有量达到一定程度以后，人们就不再增加汽车消费了。&lt;/strong&gt; 我觉得，中国很可能已经到了这个阶段。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022051809.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;为什么人们最终会减少汽车消费？&lt;/p&gt;

&lt;p&gt;&lt;ins&gt;首先，汽车是一种不划算、不环保的消费品。&lt;/ins&gt;买来以后，大多数时间停着不用；使用时占一大块道路面积，乘客往往只有一个人；每年还有高额的折旧、停车、保险、维护费用。&lt;/p&gt;

&lt;p&gt;&lt;ins&gt;其次，长时间开车的体验一点都不好。&lt;/ins&gt;连续几个小时，封闭在一个狭小的金属空间里面，不能站立和走动。精神必须高度集中，防止突发情况。&lt;/p&gt;

&lt;p&gt;&lt;ins&gt;最后，互联网减少了交通需求。&lt;/ins&gt;交流和办公都可以网上解决，需要当面完成的事情越来越少。&lt;/p&gt;

&lt;p&gt;由于以上几点，一旦有更便宜、更轻松的选择，人们就会放弃购买汽车。最近十年，公共交通、网约车、线上租车都在大发展，所以汽车销量的下滑几乎是必然的。&lt;/p&gt;

&lt;p&gt;电动车的出现，也不会改变这种趋势，因为上面几点理由依然成立。所以，我不看好汽车这个行业，我认为，不管是传统车企，还是以特斯拉为代表的新兴车企，长期的增长潜力是有限的，除非它们转变成&quot;无人驾驶服务&quot;的提供商，但这应该是很遥远的事情。&lt;/p&gt;

&lt;p&gt;而且，如果无人驾驶技术成熟，用车会极大方便，汽车的销售量将进一步下降。人类社会未来的交通解决方案，应该是公共轨道交通 + 无人驾驶的网约车网络。&lt;/p&gt;

&lt;h2&gt;非全日制计算机硕士&lt;/h2&gt;

&lt;p&gt;以前，考研录取率都在三分之一左右，三个人取一个。去年开始，下降到四分之一，今年是历年最低的24%。当然，这是全国总的统计，热门高校和热门专业的录取率远低于此。&lt;/p&gt;

&lt;p&gt;尽管考研难度越来越高，但由于就业形势不好，估计今后几年，报考人数依然只增不降，不仅应届生，很多已经工作的在职人员也会报考。&lt;/p&gt;

&lt;p&gt;但是，大家可能不知道，除了常规的全日制硕士，其实还有另一种选择，录取难度会低一些，那就是&lt;a href=&quot;https://baike.baidu.com/item/%E9%9D%9E%E5%85%A8%E6%97%A5%E5%88%B6%E7%A0%94%E7%A9%B6%E7%94%9F/2607200&quot;&gt;非全日制硕士&lt;/a&gt;。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022051817.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;所谓&quot;非全日制硕士&quot;，属于两年制专业硕士的一种，一般不脱产，允许学生在有正常工作的情况下，攻读硕士学位，上课时间通常安排在周末。&lt;/p&gt;

&lt;p&gt;你可以网上搜一下，非全日制硕士是国家正式承认的学历教育，入学也必须通过全国硕士研究生统一招生考试（全国统考），修完学分以后，可以拿到教育部认可的学历证书 + 学位证书。国家设立它的主要目的，就是为了推动在职人员培训，让那些已经在工作岗位上的人也有系统性进修的机会。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202201/bg2022011303.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;根据网上的资料，不少知名高校已经招收非全日制计算机硕士了：山东大学 、浙江大学、福州大学、西安交通大学、中国海洋大学、华中科技大学等等。&lt;/p&gt;

&lt;p&gt;这个学位很合适那些想要继续深造的在职人员，既可以获得更高学历，又不必脱产，不影响日常工作。对于应届同学，如果看重国家承认的硕士证书，但又担心考不上，也可以报考这个学位。&lt;/p&gt;

&lt;p&gt;本周有一个 &lt;strong&gt;《计算机硕士上岸集训营》&lt;/strong&gt;，就是专门为大家辅导和咨询非全日制计算机硕士考试。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202201/bg2022011305.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;h2&gt;科技动态&lt;/h2&gt;

&lt;p&gt;1、&lt;a href=&quot;https://www.nationalgeographic.com/animals/article/wildlife-watch-news-tuskless-elephants-behavior-change&quot;&gt;没有象牙的大象&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202112/bg2021122014.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;科学家在莫桑比克发现，当地的大象正在向没有象牙的方向进化。1977年到1992年，莫桑比克发生了15年内战，交战的双方都屠杀大象，用象牙去购买武器。该国4000多头大象估计被消灭了90%，剩下的都是一些没有象牙的大象。&lt;/p&gt;

&lt;p&gt;正常情况下，大约2%~4%的雌性非洲象出生时，就没有象牙。内战结束后，有51%的雌性非洲象没有象牙，而她们生出的下一代小象，雌性有高达32%没有象牙。其它偷猎严重的非洲国家，无牙大象的比例也在急剧上升。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202112/bg2021122015.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;上图的大象就没有象牙。由于象牙可以帮助大象进食，科学家非常担心，没有象牙会影响这些大象的生存。&lt;/p&gt;

&lt;p&gt;2、&lt;a href=&quot;https://m.ne21.com/news/show-170237.html&quot;&gt;弧形光伏建筑&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022051504.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;国内最大跨度的弧形光伏建筑，最近在江苏常熟落成。这是一个车间大棚，光伏板直接铺在防水支架上，替代传统彩钢瓦作为屋顶。远远看去，如同一个庞大的弧面镜。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022051505.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;该项目用掉了6.8万块光伏板，覆盖屋顶面积达17.6万平方米，最大跨度223.7米，每天可以发电十多万度。&lt;/p&gt;

&lt;p&gt;3、&lt;a href=&quot;https://www.tmtpost.com/6109686.html&quot;&gt;月球土壤种植&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022051408.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;美国佛罗里达大学的科学家，使用上个世纪阿波罗登月计划取回的月球土壤，在里面种出了阿拉伯草，证明了植物可以在月球土壤中发芽和生长，对未来的月球基地有重要意义。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022051409.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;作为对照组，科学家同时还在地球火山灰中种植。上图左边是地球火山灰，右边是月球土壤，可以看到，月球土壤的植株明显个体较小，生长速度较慢，而且植物基因也发生了一些变化。&lt;/p&gt;

&lt;p&gt;4、&lt;a href=&quot;https://www.treehugger.com/geothermal-drilling-technology-quaise-energy-5219924&quot;&gt;地热发电&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202202/bg2022022605.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;说到清洁能源，大家想到的都是水力、太阳能、风能、潮汐能等等，很少有人了解，地热也是一种清洁能源。因为地球本身就是一个巨大的核电站，我们只需要用管道，将地下的热量引导到地球表面，就能驱动涡轮机发电。&lt;/p&gt;

&lt;p&gt;以前，只有在火山或地理板块的边缘，才能获得地热能，因为那里的地热埋藏较浅。麻省理工学院孵化的一家创业公司，正在开发一种技术，能够从20公里的地下获取地热能。这样的话，就能在任何地方使用地热发电。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202202/bg2022022606.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;这需要一种前所未有的钻井技术，毕竟人类迄今最深的钻井也只有12公里深。而且，还需要价格足够便宜，低于风电和太阳能的成本。该公司宣称，他们能够做到这一点。&lt;/p&gt;

&lt;p&gt;5、&lt;a href=&quot;https://tech.hindustantimes.com/tech/news/did-nasa-mars-rover-find-a-secret-alien-doorway-find-out-the-truth-behind-this-shocking-image-71652414296727.html&quot;&gt;火星山洞&lt;/a&gt;&lt;/p&gt;

&lt;h2&gt;文摘&lt;/h2&gt;

&lt;p&gt;1、&lt;a href=&quot;https://thezbook.com/code-first-vs-product-first/&quot;&gt;代码优先与产品优先&lt;/a&gt;&lt;/p&gt;

&lt;p&gt;程序员可以分成两种：一种程序员更关心代码，另一种程序员更关心产品。&lt;/p&gt;

&lt;p&gt;我称前者为&quot;代码优先&quot;程序员，后者为&quot;产品优先&quot;程序员。&lt;/p&gt;

&lt;p&gt;&lt;img src=&quot;https://cdn.beekka.com/blogimg/asset/202205/bg2022051906.webp&quot; alt=&quot;&quot; title=&quot;&quot; /&gt;&lt;/p&gt;

&lt;p&gt;&quot;代码优先&quot;程序员痴迷于代码的架构、关注使用什么工具、库和语言，有多少测试覆盖率，诸如此类的问题。&lt;/p&gt;

&lt;p&gt;如果达到完美的代码抽象，或者使用了最新的语言功能，再或者删除了用不到的代码，&quot;代码优先&quot;程序员就会很兴奋。他们真的喜欢他们编写的代码，在他们眼里，代码本身就是目的。&lt;/p&gt;

&lt;p&gt;&quot;产品优先&quot;程序员也关心代码，但只是把它作为达到目的的一种手段。对于他们来说，代码是建筑物的脚手架和钢梁，而不是最终产品。最终产品是产品本身，而不是代码。&lt;/p&gt;

&lt;p&gt;他们认为，重要的是，产品是不是真正解决了问题？每样东西是否正常工作？人们喜欢用这个产品吗？&quot;产品优先&quot;程序员喜欢构建和发布，喜欢看到用户使用他们构建的东西。在他们眼中，产品才是目的。&lt;/p&gt;

&lt;p&gt;大型软件公司里面，你会遇到很多&quot;代码优先&quot;程序员。他们看重代码是否合乎规范，有没有拼写错误，是不是意大利面条式的代码，会不会带来技术债务。如果达不到代码审核的严谨标准，就会要求重构代码。&lt;/p&gt;

&lt;p&gt;我总是很惊讶地发现，有那么多程序员认为，&quot;代码优先&quot;就是他们追求的目标。对于项目原型，他们会问：&quot;单元测试覆盖率怎么样？&quot; 、&quot;有没有用到热门的新技术 X？&quot;、&quot;会带来很多技术债务吗？&quot;。&lt;/p&gt;

&lt;p&gt;可是，我们现在做的只是项目的原型，迟早会重写所有代码，所以上面的这些问题，在这个阶段是无关紧要的。因为我们甚至还没有认定，现在的代码就是解决问题的正确方法。&lt;/p&gt;

&lt;p&gt;&quot;代码优先&quot;程序员对编程有一个根本的误解。要知道，编程是为用户解决问题，而不是为了编写出漂亮的代码。重要的是代码能够工作，而不是代码的外观。&lt;/p&gt;

&lt;p&gt;这是否意味着，我鼓励大家编写糟糕的代码？不在乎使用什么技术或软件？&lt;/p&gt;

&lt;p&gt;绝对不是。&lt;/p&gt;

&lt;p&gt;我很在乎代码！我只是认为，我们先要保证方向正确，因为我相信，只要保证代码能够解决问题，那么通过更好的产品，最终可以获得更好的代码。反过来就不成立，更好的代码未必会获得更好的产品。&lt;/p&gt;

&lt;p&gt;我对代码的评判标准是： &lt;strong&gt;如果产品不好用，代码也不会好。&lt;/strong&gt; 好的代码只有在它产生了一个行得通的产品时才存在。&lt;/p&gt;
